# Comments

You can attach comments to any section. Comments are useful for leaving yourself notes, tracking feedback, or flagging something to revisit.

## Adding a comment

Click the comment icon (speech bubble) on any section header. A panel opens below the section with a text input. Type your note and press Enter to post it. Comments are timestamped automatically.

## Resolving a comment

Each comment has a **Resolve** action. Resolving a comment marks it as done and moves it to the resolved list. Resolved comments are hidden by default, but you can show them again via the toggle at the top of the comment panel.

To reopen a resolved comment, expand the resolved list and click the reopen action.

## Pinning a comment to a snapshot

When you are previewing a snapshot, you can pin a new comment to that snapshot. The comment will be associated with that specific version, which is useful for feedback tied to a particular draft.

## How comments are stored

Comments are stored inside the `.lyr` file and are tied to the section by ID, so reordering sections does not break them. They persist across snapshots and are included in the file if you move it to another machine.
